## UI 问题与修复

1. Next dev 首次加载在双 lockfile 工作目录下误判 Turbopack root，导致开发态扫描过大并触发资源不足。
   修复：在 `frontend/next.config.mjs` 显式设置 `turbopack.root` 为前端工作目录。
2. MCP 页首次加载在 React Strict Mode 的 effect cleanup 中显示请求中止文本。
   修复：前端忽略已主动 abort 的读取请求，不向科研用户暴露内部取消信息。
3. UI 信息架构偏长列表和卡片堆叠。
   修复：按 Google Workspace 逻辑重组分组导航、顶部搜索、资源表格、右侧详情面板、Docs/Drive/Cloud Console 式关键页面。
4. 前线轨道、电荷布居面板仍残留历史 UTF-8/GBK 误解码文本。
   修复：重写两个面板中文文案、状态标签和机制解释，并新增 `scripts/mojibake_audit.py` 接入质量门禁。
5. 中英文混排和局部 `<b>` / `font-bold` 造成文字粗细不一致。
   修复：统一中文优先字体栈，设置 `html lang="zh-CN"`，禁止浏览器合成过粗字重，并把生产组件中的突兀粗体统一为 500 权重。

## 交互体验核查

- 新增 MCP 页面仍采用中文主界面，英文仅作副标题。
- 所有 MCP tool run 都显示 provenance，页面无任意 shell 输入区。
- 生成 Prompt 和安全上传校验按钮均有 loading 禁用边界。
- MCP JSON 结果区和 prompt 预览区可滚动，不会撑破相邻卡片。
- 文献资源表格可扫描，解析质量、C级证据、warnings 和 provenance 始终可见。
- 分子库资源表可扫描，行点击后保持结构视图和分子详情面板联动。
- Gaussian 输入生成页把任务模板、参数表单和输入预览分离，避免代码预览挤压操作区。
- Gaussian 解析页明确“仅读取，不执行 Gaussian”，normalized JSON 与解析质量分区显示。
- 报告页采用章节大纲 / 报告预览 / 证据与数据来源三栏布局。
- MCP 页按 Cloud Console 逻辑展示工具表、schema、运行结果和安全边界，不再把工具卡片临时堆叠。
- 数据管理、实验-DFT 对照和量子判据引擎已清理高频路径乱码，并统一采用表格优先、证据面板和 provenance 审计逻辑。
- 整合总控台、总览驾驶舱、合并工作台、TEA 助剂、Ti 毒化、插入能量面、水解缩合、NBO 和电子云密度页已完成本轮中文化与 Google 式结构优化。
- cube 上传后显示“真实 cube”“已读取标量场”“剖切预览”等中文状态，E2E 已验证真实下采样预览可见。
- `npm run audit:mojibake` 已验证活动文件疑似乱码行数为 0；归档来源目录只记录，不参与生产构建。
- 字体巡检已验证桌面端和 390px 移动端没有 600 / 700 字重；正文为 400，标题、按钮、标签和关键数值统一为 500。

- 电子云密度页面已支持真实 cube 下采样预览；完整 Three.js marching-cubes 等值面、报告导出截图矩阵和真实大 cube 性能仍需独立巡检。
